# 音频分割工具 (网页版)

基于浏览器的音频静音分割工具，完全在客户端运行，无需服务器，可直接部署到 GitHub Pages。

## 功能

- 拖拽/点选上传音频文件（MP3、WAV、OGG、FLAC、M4A）
- 可调参数：最小静音长度、静音阈值、保留静音量
- 按静音自动分割音频，进度条 + 实时状态日志
- 分段结果在线播放、单独下载（WAV 格式）
- 一键打包下载全部片段（ZIP，支持中文文件名）
- 每个片段显示波形图
- 深色/浅色主题切换
- 响应式设计，适配桌面和移动端
- 完全纯前端，一个 HTML 文件搞定

## 使用方式

### 本地使用
直接双击 `index.html` 在浏览器中打开即可使用。

### 部署到 GitHub Pages
1. 在 GitHub 创建仓库
2. 将 `index.html` 推送到仓库
3. 启用仓库的 GitHub Pages（Settings → Pages → 选择 main 分支）
4. 访问 `https://<用户名>.github.io/<仓库名>/`

## 技术原理

所有音频处理在浏览器中完成，使用 Web Audio API：

1. `AudioContext.decodeAudioData()` 解码音频文件
2. 提取 PCM 样本，混合为单声道
3. 分帧（10ms/帧）计算 RMS 能量
4. 标记低于阈值的帧为静音
5. 连续静音超过最小长度则确定为分割点
6. 按分割点切出非静音片段（含保留静音 padding）
7. 每个片段编码为 WAV 格式（16-bit PCM）
8. 生成 Blob URL 供播放和下载
9. 手写 ZIP 打包（Store 模式 + CRC32），支持 UTF-8 文件名

## 性能

- 处理期间通过 `setTimeout` 间断让出主线程，保持 UI 响应
- 20 分钟音频文件处理时间约 3-10 秒（取决于设备）
- 所有处理在内存中完成，无网络请求

## 浏览器兼容性

- Chrome 60+
- Firefox 60+
- Safari 14+
- Edge 79+

## 注意事项

- 音频文件解码后以原始 PCM 格式驻留内存，超大文件（>100MB）建议使用原 Python 版本
- WAV 格式为无损音频，文件体积可能较大（一分钟约 5MB）
- 分割精度取决于音频本身质量和参数设置
- 输出文件名规则：`原文件名_segment_001.wav`
