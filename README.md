# 音频分割工具 (网页版)

基于浏览器音频静音检测的自动分割工具，完全在客户端运行，无需服务器。

## 功能

- 拖拽/点选上传音频文件（MP3、WAV、OGG、FLAC、M4A）
- 可调参数：最小静音长度、静音阈值、保留静音量
- 选择文件后立即显示参数，点击开始后再读取处理
- 按静音自动分割音频，进度条 + 实时状态日志
- 解码进度根据文件大小估算，混合分块处理不卡 UI
- 分段结果在线播放、单独下载（WAV 格式）
- 一键打包下载全部片段（ZIP，支持中文文件名）
- 波形图可选（默认关闭），节省资源
- 深色/浅色主题切换，自动跟随系统设置
- 桌面端左右分栏布局，手机竖屏适配
- 完全纯前端，单 HTML 文件，无外部依赖

## 使用方式

直接双击 `index.html` 在浏览器中打开即可使用。

## 技术原理

所有音频处理在浏览器中完成，使用 Web Audio API：

1. `AudioContext.decodeAudioData()` 解码音频文件
2. 分块混音为单声道（分 20 块，块间让浏览器重绘）
3. 分帧（10ms/帧）计算 RMS 能量
4. 标记低于阈值的帧为静音
5. 连续静音超过最小长度则确定为分割点
6. 按分割点切出非静音片段（含保留静音 padding）
7. 每个片段编码为 WAV 格式（16-bit PCM）
8. 生成 Blob URL 供播放和下载
9. 手写 ZIP 打包（Store 模式 + CRC32），支持 UTF-8 文件名

## 优化特性

- Web Worker 内联处理，不阻塞主线程
- Worker 进度消息节流（最多 12 次/秒）
- 回传音频数据使用 Transferable 避免内存拷贝
- 解码进度按文件大小动态估算，与全局进度条比例匹配
- 混音分 20 块异步执行，进度条同步更新
- 波形图延迟渲染 + 可开关，默认关闭
- AudioContext 复用，避免重复初始化
- Worker 崩溃自动恢复 UI 状态
- 取消操作即时重置

## 性能

- 20分钟音频（8.9MB）总处理时间约 30 秒
  - 解码约 20 秒
  - 静音检测 + 编码约 10 秒
- 所有处理在内存中完成，无网络请求
- 支持最大 500MB 文件

## 浏览器兼容性

- Chrome 76+
- Firefox 69+
- Safari 14.1+
- Edge 79+

## 注意事项

- 音频文件解码后以原始 PCM 格式驻留内存，超大文件（>100MB）建议使用桌面版处理
- WAV 格式为无损音频，文件体积可能较大（一分钟约 5MB）
- 分割精度取决于音频本身质量和参数设置
- 输出文件名规则：`原文件名-001.wav`
